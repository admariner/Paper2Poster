# Poster Defaults

Use these defaults when the user does not provide constraints.

## Dimensions

- default poster size: `48 x 36 inches`
- orientation: landscape unless venue rules say otherwise

## Layout

- `3` or `4` columns
- strongest visual near the center-left or center
- title and author strip across the top
- conclusion and QR/contact area at the bottom-right

## Copy density

- aim for `250` to `500` words of visible body copy for a first draft
- prefer bullets over paragraphs
- keep most bullets to one line when possible
- avoid more than `6` major sections

## Content priority

Order content roughly as:

1. why this matters
2. what was proposed
3. how it works
4. what results matter most
5. what the audience should remember

## Figure guidance

- one hero figure
- one compact method diagram if the paper benefits from it
- one result chart or table
- avoid tiny multi-panel figures unless the user explicitly wants dense detail

## Tone

- technical but scannable
- evidence-forward
- no marketing language unless requested

## Style hints for `poster.yaml`

If asked to draft a style config, bias toward:

- high contrast headers
- generous whitespace
- consistent accent color
- readable sans-serif body text
- minimal decorative elements
